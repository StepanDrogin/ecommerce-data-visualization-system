# Архитектура приложения

Система строится как монорепозиторий с разделением на frontend, backend и общий пакет типов.

## Компоненты

- `apps/frontend` - пользовательский интерфейс аналитического дашборда.
- `apps/backend` - REST API для данных о товарах, заказах и аналитике.
- `packages/shared` - общие типы, используемые обеими частями приложения.
- `PostgreSQL` - основное хранилище данных.
- `Redis` - слой кэширования для аналитических запросов.

## Поток данных

```text
React Dashboard -> REST API -> NestJS Services -> Prisma -> PostgreSQL
                                      |
                                      -> Redis cache
```

На стартовом этапе backend возвращает демонстрационные данные, чтобы зафиксировать API-контракты и структуру интерфейса.
